Push dulu yang lokal, di PC ini

PC Host

VM nanti clone dari GitHub, bukan dari folder ini — jadi apa pun yang belum ke-push dianggap tidak ada oleh VM.

Cek dulu apa yang udah ke-commit vs yang masih nyangkut di working tree
git status
# kalau ada yang nggak terduga muncul, commit atau stash dulu sebelum lanjut —
# latihan ini harus nge-clone kode yang sama persis kayak yang beneran mau kamu
# kasih ke production
Push
git push origin main
Catatan

Fix origin CORS yang barusan di sesi ini itu cuma buat config dev lokal (localhost) — nggak perlu ikut ke VM. CORS production nanti dikonfigurasi ulang dari nol di Fase 5, disesuaikan sama alamat yang beneran dijawab VM-nya.

1

Buat VM-nya

VirtualBox

New VM → kasih nama misalnya mbrs-office-server, tipe Microsoft Windows, versi Windows 11 (64-bit).

SettingNilaiKenapa
MemoryMinimal 4096 MBApache + MySQL + Reverb + queue worker jalan bareng — minimum di README sendiri 2 GB, kasih ruang lebih
Processor2 coreWindows 11 sendiri minta 2+
Disk64 GB, dynamically allocatedWindows 11 doang udah ~20 GB sebelum instal apa-apa
Network AdapterBridged Adapter, bukan NATVM dapet IP sendiri di jaringan LAN — bisa diakses dari PC host dan mesin lain di jaringan, persis kayak server kantor beneran
Gotcha

Aktifkan Nested VT-x/AMD-V di System → Processor VM kalau installer Windows 11 nolak boot karena cek TPM/Secure Boot — atau pakai varian ISO Windows 11 yang skip requirement TPM buat VM.

2

Instal Windows 11 Pro

~20 menit, kebanyakan nunggu

Walkthrough installer standar. Dua pilihan yang ngaruh ke langkah berikutnya:

  • Pilih "Set up for work or school" / akun lokal aja cukup — VM ini nggak perlu akun Microsoft.
  • Setelah boot pertama, instal VirtualBox Guest Additions (menu Devices → Insert Guest Additions CD) — dapet clipboard bersama dan window yang bisa di-resize, dua-duanya bikin fase berikutnya jauh lebih nggak ribet.
3

Instal stack-nya, di dalam VM

Guest OS

Semua yang ada di tabel requirement README. Download tiap installer lewat browser di dalam VM (bukan di host lalu di-copy) — lebih gampang kalau installer-nya juga ke-cache di dalam guest, jaga-jaga kalau nanti butuh reinstall.

1 — XAMPP (PHP 8.2+, Apache, MySQL)

Buka apachefriends.org/download.html. Halaman utama cuma nawarin versi PHP terbaru — kalau itu bukan 8.2.x, klik link "other versions" di bawah tombol download dan cari XAMPP yang bawaannya PHP 8.2.x. Ini harus persis sama requirement composer.json yaitu ^8.2 — PHP bawaan yang lebih baru/lama itu penyebab paling umum bug membingungkan pas pindah mesin.

  • Jalankan installer-nya. Kalau Windows Defender / antivirus nge-warn "unknown publisher", itu normal untuk installer XAMPP — pilih "Run anyway".
  • Di layar pemilihan komponen, cukup centang Apache, MySQL, PHP, dan phpMyAdmin — FileZilla/Mercury Mail/Tomcat nggak kepakai, bisa di-uncheck biar instalasi lebih cepat.
  • Install ke lokasi default C:\xampp — path lain bisa dipakai tapi semua contoh command di panduan ini asumsi default.
  • Kalau muncul warning soal UAC/folder Program Files, klik lanjut — XAMPP memang lebih rewel kalau diinstal di luar root C:\.
Nyalain servicenya & cek dulu dari XAMPP Control Panel

Buka XAMPP Control Panel dari Start Menu, klik Start di baris Apache dan MySQL — dua-duanya harus jadi hijau. Kalau Apache gagal start (port 80 bentrok), nggak masalah untuk latihan ini karena nanti dipindah ke port 8000/8001 di Fase 7 — abaikan dulu, fokus MySQL-nya harus hijau karena dibutuhkan buat migrate database.

Tambahin PHP ke PATH manual

Installer XAMPP tidak otomatis nambahin php.exe ke PATH Windows — beda dari Node/Git yang biasanya otomatis. Harus manual: Search Windows → "Edit the system environment variables" → Environment Variables → System variables → Path → Edit → New, tambahin C:\xampp\php, lalu OK di semua dialog.

# tutup semua jendela terminal yang lagi kebuka, buka yang baru (PATH baru
# nggak kepakai di terminal yang udah lebih dulu kebuka), lalu:
php -v
# pastikan yang keluar 8.2.x
Sekalian

Buka http://localhost/phpmyadmin (setelah Apache/MySQL hijau) dan buat database kosong sekarang, misalnya mbrs_db — biar Fase 5 (php artisan migrate) tinggal isi tabel ke database yang udah ada.

2 — Composer

Buka getcomposer.org/download, download Composer-Setup.exe (installer Windows, bukan versi manual/command-line di halaman yang sama).

  • Installer bakal otomatis nyari php.exe — pastikan yang kedeteksi adalah C:\xampp\php\php.exe (kalau ada lebih dari satu PHP di mesin, pilih yang ini secara manual di step itu).
  • Biarin opsi proxy kosong kecuali kantor/jaringan kamu emang pakai proxy.
  • Lanjut sampai selesai dengan pengaturan default.
# buka terminal baru lagi setelah instalasi selesai:
composer -V
3 — Node.js 20.x LTS

Buka nodejs.org — di halaman utama biasanya ada dua tombol download, pilih yang berlabel LTS (bukan "Current"), pastikan versinya baris 20.x sesuai README (kalau LTS yang lagi aktif sudah naik ke versi lain, buka nodejs.org/en/download dan pilih 20.x dari dropdown versi).

  • Jalankan .msi-nya, next-next dengan default — installer ini otomatis nambahin node dan npm ke PATH.
  • Di langkah "Tools for Native Modules", centang boks itu boleh di-skip (uncheck) — project ini nggak butuh native module yang perlu dikompilasi.
# terminal baru lagi:
node -v   # harus v20.x
npm -v
4 — Git for Windows

Buka git-scm.com/download/win, download installer 64-bit-nya.

  • Di layar "Adjusting your PATH environment", pastikan opsi yang kepilih adalah "Git from the command line and also from 3rd-party software" (opsi default/recommended) — bukan yang "Git Bash only".
  • Di layar "Choosing the default editor", boleh dibiarin default (Vim) atau pilih Notepad kalau belum kenal Vim — nggak ngaruh ke langkah manapun di panduan ini.
  • Layar "line ending conversions": biarin default "Checkout Windows-style, commit Unix-style line endings".
  • Sisanya next-next default sampai selesai.
# terminal baru lagi:
git --version
Gotcha — kena berkali-kali kalau kelewat

Empat installer di atas semuanya cuma nulis ke PATH Windows saat itu juga — jendela terminal/PowerShell yang udah kebuka duluan tetap pakai PATH lama sampai ditutup dan dibuka ulang. Kalau abis instal composer/node/git terus command-nya masih "not recognized", coba tutup terminal dan buka yang baru dulu sebelum curiga instalasi gagal.

Gotcha

Windows Defender Firewall bakal diam-diam nge-block koneksi masuk ke Apache/Reverb dari mesin lain di jaringan sampai kamu izinin. Lakukan sekali sekarang: Control Panel → Windows Defender Firewall → Allow an app → izinkan httpd.exe (Apache) di jaringan Private, dan nanti buka juga port Reverb secara eksplisit (Fase 8).

Konfigurasi & migrate backend

server/
Instal dependency
cd server
composer install --no-dev --optimize-autoloader
Environment
copy .env.example .env
php artisan key:generate
php artisan reverb:install   # generate REVERB_APP_ID / KEY / SECRET ke .env

Lalu edit .env manual:

KeyIsi dengan
APP_ENVproduction
APP_DEBUGfalse
APP_URLhttp://<IP LAN VM>:8000 — cek pakai ipconfig di dalam VM
DB_CONNECTIONmysql + database asli yang kamu buat di phpMyAdmin
BROADCAST_CONNECTIONreverb
Migrate
php artisan migrate --force
php artisan storage:link
Gotcha — ini bakal kena

Begitu frontend disajikan dari origin yang beda (hasil build Fase 6, atau IP VM lewat Apache), server/config/cors.php bagian allowed_origins harus nyertain origin itu persis — pelajaran yang sama kayak masalah localhost:4173 tadi. Tambahin dulu sebelum tes login, jangan nunggu gagal dulu baru sadar.

7

Arahkan Apache ke dua-duanya

XAMPP Apache

Ada dua hal yang butuh "rumah": API Laravel (server/public) dan frontend statis (client/dist). Paling gampang buat latihan: dua port di Apache yang sama.

Edit xampp/apache/conf/extra/httpd-vhosts.conf
<VirtualHost *:8000>
    DocumentRoot "C:/xampp/htdocs/mbrs-v1/server/public"
    <Directory "C:/xampp/htdocs/mbrs-v1/server/public">
        AllowOverride All
        Require all granted
    </Directory>
</VirtualHost>

<VirtualHost *:8001>
    DocumentRoot "C:/xampp/htdocs/mbrs-v1/client/dist"
    <Directory "C:/xampp/htdocs/mbrs-v1/client/dist">
        AllowOverride All
        Require all granted
        FallbackResource /index.html
    </Directory>
</VirtualHost>
Tambahin Listen directive-nya, lalu restart Apache
# di httpd.conf, di samping "Listen 80" yang udah ada:
Listen 8000
Listen 8001
Kenapa FallbackResource

Ini SPA React Router — tanpa ini, refresh di route selain / bakal 404, karena Apache nyari file asli di path itu.

8

Jaga Reverb & queue tetap hidup

Proses background

Ini bukan request/response biasa — harus tetap jalan terus. README merekomendasikan Supervisor (Linux) atau NSSM (Windows). Buat latihan, 2 jendela terminal kebuka aja udah cukup:

rem terminal 1
cd C:\xampp\htdocs\mbrs-v1\server
php artisan reverb:start

rem terminal 2
php artisan queue:work --tries=1
Gotcha

Port Reverb (default 8080) butuh allow-rule Windows Firewall yang sama kayak Apache kalau mau mesin lain di jaringan lihat update realtime — kalau nggak, app-nya tetap jalan, cuma kerasa "statis" (nggak auto-refresh) buat siapa pun yang connect dari luar VM.

Kalau mau ini tetap jalan setelah restart tanpa buka terminal manual lagi

Itu fungsinya NSSM — bungkus tiap perintah jadi Windows Service beneran. Worth dilakukan setelah kamu konfirmasi semuanya jalan manual, jangan sebelum itu.

9

Verifikasi — dari PC host, bukan cuma di dalam VM

Tes yang sebenarnya

Inti dari jaringan Bridged: perlakukan VM kayak mesin di ruangan sebelah.

  • Di dalam VM, jalankan ipconfig dan catat alamat IPv4-nya (kira-kira 192.168.1.x).
  • Dari browser PC host ini sendiri — bukan browser di dalam VM — buka http://<IP tadi>:8001.
  • Login. Kalau gagal, buka DevTools → Network dulu, bukan log backend — penolakan CORS dan error 500 asli itu kelihatan identik dari sisi form login.
  • Buat booking, pastikan langsung muncul tanpa refresh (Reverb beneran nyampe ke kamu lintas jaringan, bukan cuma di dalam VM).

Ciri-ciri "beres"

Browser di PC host ini, akses IP LAN VM, login, booking ruangan, dan langsung muncul live — semua tanpa perlu balik lagi ke layar VM-nya. Itu bentuk yang sama kayak rollout kantor beneran: meja seseorang, server orang lain.
