Cek prasyarat dulu

Sekali saja

Kalau XAMPP di mesin ini sudah pernah dipakai buat project lain, jangan asumsikan versinya cocok — cek satu-satu.

KomponenMinimumCek dengan
PHP (bawaan XAMPP)8.2.xphp -v
Composer2.xcomposer -V
Node.js20.x LTSnode -v
Gitberapa saja yang cukup barugit --version
MySQL (XAMPP)aktif via XAMPP Control Panel — atau pakai SQLite kalau cuma butuh dev cepatXAMPP Control Panel → baris MySQL hijau
Gotcha paling sering

PHP yang kepakai harus persis 8.2.x, sesuai composer.json (^8.2). Kalau php -v nunjukin versi lain — XAMPP terinstal lebih dari satu, atau ada PHP sistem lain duluan di PATH — composer install nanti bisa gagal diam-diam atau jalan tapi error runtime yang membingungkan pas dites. Betulkan urutan PATH atau reinstall XAMPP versi PHP 8.2 dulu sebelum lanjut.

1

Clone project ke htdocs

Terminal
cd C:\XAMPP\htdocs
git clone https://github.com/jodigpaksi/mbrs-v1.git
cd mbrs-v1

Kalau folder ini sudah ada isinya (lagi baca panduan ini dari clone yang sudah jalan), lewati fase ini dan langsung ke Fase 2 — cukup pastikan git status bersih dulu sebelum composer install/npm install ulang.

2

Install dependency backend

server/
cd server
composer install

Untuk dev lokal pakai composer install biasa (bukan --no-dev) — butuh Tinker dan Pail buat testing/debugging sehari-hari.

3

Konfigurasi environment backend

server/.env
Buat file .env dari template, generate app key
copy .env.example .env
php artisan key:generate
Pilih database
OpsiSetting di .envKapan pakai
SQLiteDB_CONNECTION=sqlite, kosongkan DB_HOST/DB_DATABASE/dllpaling cepat buat coba-coba, tidak perlu buka phpMyAdmin
MySQL (XAMPP)DB_CONNECTION=mysql + DB_HOST=127.0.0.1, DB_DATABASE, DB_USERNAME=root, DB_PASSWORD= (kosong, default XAMPP)kalau mau kondisi mendekati production, atau perlu buka data lewat phpMyAdmin

Kalau pilih SQLite, buat dulu file kosongnya sebelum migrate:

# dari folder server/
type nul > database\database.sqlite

Kalau pilih MySQL, buat databasenya dulu lewat http://localhost/phpmyadmin — nama bebas, samakan dengan DB_DATABASE di .env.

Generate kredensial Reverb (WebSocket realtime)
php artisan reverb:install

Ini otomatis ngisi REVERB_APP_ID, REVERB_APP_KEY, REVERB_APP_SECRET di .env. Pastikan juga baris ini ada/benar:

BROADCAST_CONNECTION=reverb
REVERB_HOST="localhost"
REVERB_PORT=8080
REVERB_SCHEME=http
Catatan

server/.env dan client/.env memang tidak pernah ikut ke git (sudah di-.gitignore) — jadi wajar kalau clone baru selalu mulai dari .env.example dan isi ulang sendiri, bukan tanda ada yang salah.

4

Migrasi database

server/
php artisan migrate

Kalau mau langsung ada data contoh (user/ruangan/gedung dummy) buat dicoba-coba, tambahkan --seed. Cek dulu isi database/seeders/ kalau ragu seeder mana yang jalan.

5

Install dependency frontend

client/
cd ..\client
npm install
copy .env.example .env

Edit client/.env, isi VITE_REVERB_APP_KEY supaya sama persis dengan REVERB_APP_KEY di server/.env (hasil Fase 3). VITE_REVERB_HOST/PORT/SCHEME biasanya sudah cocok dengan default di .env.example.

Gotcha

Client tidak lewat proxy Vite ke API — axios.ts langsung memanggil http://localhost:8000/api. Kalau nanti API dijalankan di port lain, itu satu-satunya tempat yang perlu disesuaikan, bukan config Vite.

6

Jalankan semua service

2 terminal

Reverb tidak ikut dalam composer run dev — harus dijalankan terpisah, di terminalnya sendiri, kalau mau fitur realtime (booking auto-muncul tanpa refresh) benar-benar jalan.

Terminal 1 — server, queue, log viewer, Vite sekaligus
# dari folder server/
composer run dev
Terminal 2 — Reverb (proses terpisah, wajib)
# dari folder server/, terminal baru
php artisan reverb:start
Kenapa dipisah

composer run dev menjalankan php artisan serve + queue:listen + pail (log viewer) + npm run dev secara bersamaan lewat satu perintah — tapi Reverb sengaja tidak dimasukkan ke script itu, jadi kalau lupa buka terminal kedua, aplikasi tetap jalan normal, cuma update booking baru kerasa setelah refresh manual (tidak realtime).

7

Verifikasi

Browser
  • Buka http://localhost:5173 (URL yang ditampilkan Vite) — harus muncul halaman login, bukan error koneksi.
  • Login pakai user hasil migrate --seed, atau buat user baru lewat php artisan tinker kalau belum ada seeder.
  • Buat satu booking, lalu buka tab browser kedua yang juga login — booking harus langsung muncul di tab kedua tanpa refresh. Kalau tidak, cek lagi Terminal 2 (Reverb) masih hidup.
  • API bisa dites langsung lewat http://127.0.0.1:8000/api/... — ini yang dipanggil axios.ts di balik layar.
8

Kalau ada yang error

Troubleshooting singkat
GejalaPenyebab yang paling mungkin

Booking baru tidak auto-muncul di client lainTerminal Reverb (php artisan reverb:start) belum dijalankan atau sudah mati
Foto/logo yang diupload jadi 404Symlink storage belum dibuat — jalankan php artisan storage:link dari server/
composer install sukses tapi ada error runtime anehPHP yang dipakai Apache/CLI bukan 8.2.x — cek ulang Fase 0
Reminder email / anti-ghost tidak pernah jalan otomatisWajar untuk dev — dua command itu (bookings:send-reminders, bookings:release-ghosts) hanya jalan lewat scheduler (schedule:run tiap menit), yang di lokal biasanya tidak diaktifkan; jalankan manual saat perlu dites: php artisan bookings:send-reminders

Ciri-ciri "beres"

Dua terminal jalan (composer run dev + reverb:start), browser di localhost:5173 bisa login, dan booking baru muncul realtime di tab lain tanpa refresh. Kalau mau lihat versi yang mensimulasikan mesin server kantor beneran, lanjut ke vm-deploy-guide_id.html.
