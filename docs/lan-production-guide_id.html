Ringkasan & prasyarat

Baca dulu

Ada 3 komponen yang jalan terpisah, dan cuma 1 yang butuh dibangun ulang tiap ada perubahan kode.

KomponenSebelum (dev)Sesudah (LAN)
API Laravelphp artisan serve (localhost saja)Apache vhost port 8000, DocumentRoot server/public
Client ReactVite dev server :5173 (localhost saja)Apache vhost port 8002, serve client/dist hasil build
Reverb (realtime)php artisan reverb:startsama persis — sudah bind 0.0.0.0 secara default, tidak perlu Apache
Prasyarat

Device lain (HP/laptop) harus terhubung ke WiFi/jaringan yang sama persis dengan PC ini. IP jenis 192.168.x.x tidak bisa diakses dari luar jaringan lokal (internet) tanpa setup tambahan seperti port forwarding — di luar cakupan panduan ini.

1

Cek IP LAN PC ini

PowerShell
ipconfig | Select-String "IPv4"

Kalau muncul lebih dari satu baris, pilih yang cocok sama adapter WiFi/Ethernet fisik. Contoh di panduan ini pakai 192.168.8.19 — ganti semua kemunculan IP itu dengan IP milikmu sendiri.

Gotcha

Adapter virtual (VirtualBox, VPN, Hyper-V) sering ikut muncul juga, misalnya 192.168.56.1 — itu bukan IP yang bisa diakses device lain di jaringan rumah/kantor. Kalau ragu IP mana yang benar, cek juga lewat ipconfig /all dan cocokkan nama adapternya (Wi-Fi / Ethernet vs VirtualBox Host-Only).

IP bisa berubah

Kalau router pakai DHCP biasa, IP PC ini bisa berubah tiap reconnect/reboot. Kalau setup ini sering dipakai, aktifkan DHCP reservation / IP statis untuk PC ini dari halaman admin router, supaya tidak perlu ulang Fase 4 setiap IP berubah.

2

Build client jadi static

client/
cd client
npm run build

Ini menjalankan tsc -b && vite build, hasilnya folder client/dist — kumpulan file statis (HTML/JS/CSS) yang nanti di-serve langsung oleh Apache, bukan lagi lewat Vite dev server.

Ingat ini

client/dist adalah hasil "beku" dari kode saat build dijalankan. Tiap ada perubahan di client/src yang mau kelihatan di versi LAN (:8002), wajib npm run build ulang — dev server npm run dev di :5173 tidak otomatis sinkron ke dist.

3

Tambah vhost Apache

httpd-vhosts.conf

Edit C:\XAMPP\apache\conf\extra\httpd-vhosts.conf, tambahkan dua blok ini. Ganti port kalau sudah dipakai project lain (cek dulu blok existing di file yang sama).

Vhost API Laravel — port 8000
Listen 8000
<VirtualHost *:8000>
    DocumentRoot "C:/XAMPP/htdocs/mbrs-v1/server/public"
    <Directory "C:/XAMPP/htdocs/mbrs-v1/server/public">
        AllowOverride All
        Require all granted
    </Directory>
    ErrorLog "logs/mbrs-v1-api-error.log"
    CustomLog "logs/mbrs-v1-api-access.log" common
</VirtualHost>
Vhost client React (static build) — port 8002
Listen 8002
<VirtualHost *:8002>
    DocumentRoot "C:/XAMPP/htdocs/mbrs-v1/client/dist"
    <Directory "C:/XAMPP/htdocs/mbrs-v1/client/dist">
        AllowOverride All
        Require all granted
        # React Router (BrowserRouter) fallback — path yang bukan file
        # fisik dilempar balik ke index.html biar client-side routing jalan
        FallbackResource /index.html
    </Directory>
    ErrorLog "logs/mbrs-v1-client-error.log"
    CustomLog "logs/mbrs-v1-client-access.log" common
</VirtualHost>
Test config sebelum restart (wajib!)
C:\xampp\apache\bin\httpd.exe -t

Harus keluar Syntax OK. Kalau ada typo di vhost dan langsung restart tanpa test dulu, Apache bisa gagal nyala total — termasuk buat project lain yang numpang di XAMPP yang sama.

Kenapa Apache, bukan artisan serve

php artisan serve itu proses manual yang harus kamu jalankan tiap buka terminal, dan defaultnya cuma bind ke localhost. Apache dari XAMPP sudah otomatis nyala terus (lewat XAMPP Control Panel) dan bind ke semua network interface — jadi begitu vhost ditambahkan, API otomatis bisa diakses dari IP LAN tanpa proses tambahan yang perlu diingat-ingat tiap hari.

4

Sesuaikan .env & CORS

4 file

Semua ini defaultnya hardcode ke localhost — ganti ke IP LAN PC ini (dari Fase 1) di 4 tempat berikut.

server/.env
APP_URL=http://192.168.8.19:8000
FRONTEND_URL=http://192.168.8.19:8002

FRONTEND_URL dipakai di Booking.php buat bikin link di email (reminder, anti-ghost, dll) — kalau tidak diganti, link di email tetap nunjuk ke localhost dan tidak bisa dibuka dari device lain.

server/config/cors.php
'allowed_origins' => [
    'http://localhost:5173',
    'http://localhost:5174',
    'http://192.168.8.19:8002',
],

Tanpa ini, semua request dari client versi LAN akan ditolak browser dengan CORS error, walaupun API-nya sendiri hidup dan bisa diakses langsung.

client/src/api/axios.ts
const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL ?? 'http://localhost:8000/api',
  ...
})

Sebelumnya baseURL hardcode string 'http://localhost:8000/api'. Diubah supaya baca dari environment variable, dengan fallback ke localhost kalau env-nya kosong (tetap aman buat dev biasa).

client/.env
VITE_API_URL=http://192.168.8.19:8000/api
VITE_REVERB_APP_KEY=<samakan dengan REVERB_APP_KEY di server/.env>
VITE_REVERB_HOST=192.168.8.19
VITE_REVERB_PORT=8080
VITE_REVERB_SCHEME=http

Karena ini file .env yang sama dipakai baik oleh npm run dev maupun npm run build, dan IP LAN tetap bisa diakses dari PC ini sendiri (loopback ke diri sendiri), mengganti nilainya ke IP LAN tidak merusak dev mode biasa di :5173.

Jangan lupa

Setelah ubah client/.env atau axios.ts, build ulang (npm run build) supaya perubahan kepakai di versi :8002 — env var Vite di-inline saat build, bukan dibaca ulang saat runtime seperti di backend.

5

Restart Apache

Ambil efek vhost baru
Lewat XAMPP Control Panel (cara paling aman)

Klik Stop lalu Start di baris Apache. Ini cara yang paling konsisten di mesin ini.

Lewat PowerShell (kalau Control Panel tidak kebuka)
# Apache di XAMPP ini TIDAK terdaftar sebagai Windows Service —
# apache_stop.bat/apache_start.bat kadang macet nunggu "Press any key"
# kalau dijalankan non-interaktif. Cara paling reliable:
Get-Process httpd | Stop-Process -Force
Start-Process "C:\xampp\apache\bin\httpd.exe" -WorkingDirectory "C:\xampp\apache\bin" -WindowStyle Hidden
Verifikasi semua port kebuka
Get-NetTCPConnection -LocalPort 80,8000,8002 -State Listen

Harus muncul ketiga port itu dengan status Listen.

6

Buka Windows Firewall

PowerShell as Administrator

Tanpa ini, PC sendiri tetap bisa akses (lewat localhost/IP sendiri), tapi device lain akan kena timeout — Windows Firewall diam-diam blok koneksi masuk ke port custom dari luar.

New-NetFirewallRule -DisplayName "mbrs-v1 API (8000)" -Direction Inbound -Protocol TCP -LocalPort 8000 -Action Allow
New-NetFirewallRule -DisplayName "mbrs-v1 Client (8002)" -Direction Inbound -Protocol TCP -LocalPort 8002 -Action Allow
New-NetFirewallRule -DisplayName "mbrs-v1 Reverb (8080)" -Direction Inbound -Protocol TCP -LocalPort 8080 -Action Allow
Harus Administrator

New-NetFirewallRule gagal dengan pesan Access is denied kalau PowerShell tidak dijalankan sebagai Administrator. Klik kanan PowerShell → Run as Administrator dulu.

7

Reverb & queue worker

Proses terpisah, tetap manual

Dua proses ini tidak ikut lewat Apache — tetap harus dijalankan manual di terminal masing-masing, persis seperti dev biasa.

# dari server/, biarkan terminal ini tetap terbuka
php artisan reverb:start

# kalau butuh email/notifikasi terkirim (queue jobs), terminal lain:
php artisan queue:listen --tries=1 --timeout=0
Sudah otomatis benar

config/reverb.php defaultnya REVERB_SERVER_HOST=0.0.0.0 — jadi Reverb otomatis bisa diakses dari LAN tanpa perlu flag tambahan, cukup pastikan firewall port 8080 (Fase 6) sudah kebuka.

8

Verifikasi dari device lain

HP / laptop lain
  • Sambungkan device ke WiFi yang sama persis dengan PC ini.
  • Buka http://192.168.8.19:8002 di browser device tersebut — harus muncul halaman login.
  • Login, lalu coba buat/lihat booking — kalau ini jalan, berarti koneksi ke API (:8000) dan CORS sudah benar.
  • Buka dua device sekaligus, buat booking di salah satu — device lain harus update otomatis tanpa refresh kalau Reverb (:8080) sudah kebuka dan jalan.
9

Balik ke mode dev sehari-hari

Tidak berubah banyak

Setelah setup di atas selesai sekali, workflow dev harian tidak berubah — cuma satu proses yang hilang.

Frontend — tetap seperti biasa
# dari client/
npm run dev

Buka localhost:5173 seperti biasa. API call otomatis kena VITE_API_URL yang sudah nunjuk ke Apache (:8000) — jadi tidak perlu php artisan serve lagi, karena Apache selalu hidup di background.

Backend — cuma queue & log viewer
# dari server/
composer run dev
Bug lama yang sudah dibenerin

Sebelumnya composer run dev juga mencoba menjalankan npm run dev dari folder server/ — tapi server/package.json itu sisa skeleton default Laravel yang tidak pernah dipakai (tidak ada Blade view, server/node_modules tidak pernah di-install), jadi step itu selalu error dan (karena flag --kill-others) bisa ikut mematikan proses queue/pail juga. Sekarang composer run dev cuma menjalankan queue:listen + pail. Frontend tetap dijalankan terpisah lewat npm run dev di client/, seperti kebiasaan sebelumnya — tidak ada yang berubah di sana.

Ciri-ciri "beres"

Firewall sudah dibuka, http://192.168.8.19:8002 bisa dibuka & dipakai login-booking dari HP/laptop lain di jaringan yang sama, dan realtime sync antar device jalan tanpa refresh manual. Dev harian tetap npm run dev di client/ + composer run dev di server/, seperti sebelumnya.
